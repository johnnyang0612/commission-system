# 業務洽談管理系統 PRD

**版本：** v1.0  
**建立日期：** 2025-08-15  
**負責單位：** 川輝科技

## 1. 系統目標

打造一套可集中管理所有洽談中案子的系統，讓業務能快速判斷「今天該追誰」、「誰能成交」、「哪些案子先冷凍」，並提供完整的追蹤閉環與數據分析能力，協助主管與財務即時掌握 pipeline 狀況。

## 2. 主要使用者角色

- **業務：** 快速輸入案件、追蹤狀態、安排行動、掌握優先順序
- **業務主管：** 檢視團隊 pipeline、指導卡案解法、調整資源分配
- **財務：** 根據預計簽約日與機會金額做現金流預測
- **老闆：** 即時看到成交進度與高價案狀況

## 3. 功能模組與欄位設計

| 分潤比例 | 數字% | ✅ | 用於獎金計算 |
| 負責人 | 下拉 | ✅ | 指派業務 |
| 洽談階段 | 下拉 | ✅ | 初談 / 提案 / 報價 / 談判 / 待簽約 |
| 預計簽約日 | 日期 | ✅ | 用於 pipeline 預測 |
| 客戶來源 | 文字 / 下拉 | - | 網路行銷 / 轉介紹 / 陌生開發 |

| 欄位 | 類型 | 必填 | 說明 |

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|

| 預計收款月份 | 月份選擇 | - | 預估現金流 |
| 機會金額 | 系統自動 | 系統 | 預估金額 × 成交率% |

## 4. 排序與提醒規則

### 案件列表預設排序
成交率高 > 下次追蹤日期近 > 預估金額大

### 提醒機制
- 下次追蹤日期到 → 系統通知業務 + 主管
- 距離簽約日 7 天 → 系統提醒跟進
- 14 天未更新 → 系統黃牌提示
- 高價案（例如 >50萬）+ 高成交率 → 自動列入優先池

## 5. 權限管理

- **業務：** 可編輯自己負責的案件
- **主管：** 可檢視、編輯所有業務案件
- **財務：** 僅檢視財務相關欄位與 pipeline 報表
- **老闆：** 全權檢視

## 6. 報表與分析

- **Pipeline 總覽：** 依成交率、金額、簽約日顯示預測營收
- **來源轉換率：** 不同客戶來源的成交率分析
- **阻力分析：** 不同阻力類型對成交率的影響
- **行動效率分析：** 行動紀錄數 vs 成交率
