## 1. 概覽

川輝AI助理是川輝科技的內部智慧工作助手，提供兩個使用入口：

| 入口 | 位置 | 誰能用 |
|------|------|--------|
| **LINE 群組** | 在 LINE 群組中直接下指令 | 群組內的公司員工 |
| **網頁版 AI Chat** | 系統內 `/ai-chat` 頁面 | 所有登入的系統使用者 |

兩個入口共用同一套 AI 核心，能力相通。

## 2. LINE 群組指令

### 2.1 建立會議

建立 Google Calendar 事件 + Google Meet 連結，自動寄出日曆邀請。

**指令格式（任一種都行）：**

```
/建立會議 4/15 15:00~16:00 所有人
/建立會議 明天 下午3點
/會議 下周三 10:00 王經理 李專員
```

**也支援自然語言（不需要嚴格格式）：**

```
幫我建一個明天下午三點的會議
建個會議 禮拜五早上10點 跟王經理聊
約明天 3 點開會 大家都來
排一個下周二的會議 9 點半
```

**支援的日期寫法：**

| 寫法 | 意思 |
|------|------|
| `4/15`、`04/15` | 4 月 15 日 |
| `2026-04-15` | 2026 年 4 月 15 日 |
| `明天`、`後天`、`大後天` | 相對日期 |
| `今天` | 今天 |
| `下周一` ~ `下周日` | 下週的某天 |
| `這周三`、`周五` | 本週的某天 |
| `下禮拜二` | 同「下周二」 |

**支援的時間寫法：**

| 寫法 | 意思 |
|------|------|
| `15:00` | 下午 3 點整 |
| `15:00~16:00` | 3 點到 4 點 |
| `下午3點` | 15:00 |
| `早上9點半` | 09:30 |
| `3點30分` | 15:30（上下文判斷上/下午） |

**支援的參與者寫法：**

| 寫法 | 意思 |
|------|------|
| `所有人`、`大家`、`全部` | 群組內所有員工 |
| `王經理 李專員` | 指定人員（空格分隔） |
| 不寫 | 預設「所有人」 |

**建立後系統會：**
1. 在你的 Google Calendar 建立事件（用管理員帳號）
2. 自動產生 Google Meet 連結
3. 所有有 Email 的參與者收到日曆邀請
4. 在 LINE 群組回覆會議資訊與連結
5. 自動排程提醒（前一天 + 前一小時）

---

### 2.2 修改會議

**指令格式：**

```
/修改會議 週三會議 改時間 4/16 14:00
/修改會議 客戶會議 改標題 需求確認會議
```

**也支援自然語言：**

```
把明天的會議延後一小時
會議改到後天同一時間
修改會議時間 改成下午兩點半
```

**修改後系統會：**
1. 更新資料庫中的會議記錄
2. 同步更新 Google Calendar 事件
3. 受邀者會收到更新通知
4. 在 LINE 群組回覆修改結果
5. 重設提醒時間

---

### 2.3 取消會議

**指令格式：**

```
/取消會議 週三會議
/取消會議 客戶會議 原因：客戶行程衝突
```

**也支援自然語言：**

```
取消明天的會議
明天會議不開了
```

**取消後系統會：**
1. 刪除 Google Calendar 事件
2. 受邀者會收到取消通知
3. 刪除排程提醒
4. 在 LINE 群組回覆取消確認

---

### 2.4 綁定 Email

讓系統知道群組成員的 Email，之後建立會議就能自動寄日曆邀請。

**綁定自己的：**
```
/綁定 andy@company.com
```

**幫別人綁（需管理者權限）：**
```
/綁定 @王經理 wang@company.com
@川輝AI助理 @客戶A 綁定Email:client@gmail.com
```

### 2.5 知識問答

問關於群組過去討論過的事情，AI 會根據歷史訊息、會議記錄、專案資料回答。

**使用方式：**
```
@川輝AI助理 上次討論的結論是什麼？
@川輝AI助理 這個專案目前卡在哪裡？
/問 上次會議的待辦事項
/問 王經理之前說的報價是多少？
```

**AI 會搜尋的資料來源：**
- 群組近 100 則訊息
- 關聯的會議記錄（摘要、決議、待辦）
- 關聯的專案/商機資訊
- AI 自動摘要

**注意：** 回答會標注資訊來源（日期或文件名稱），不會瞎編。找不到就會說「找不到」。

---

### 2.6 取回過期檔案

LINE 的檔案連結 7 天後會過期，但系統已經備份。可以這樣取回：

1. **長按**群組裡那則已過期的檔案訊息
2. 選「**回覆**」
3. 輸入 `取回` 或 `取檔` 或 `找檔`

```
（回覆某則檔案訊息）
取回
```

系統會自動找到備份，回傳下載連結。

---

### 2.7 群組設定

設定群組的負責人、關聯的商機或專案。

```
/設定 負責人=王小明 洽談=ABC公司
/設定 專案=PRJ001
/設定 類型=prospect
/設定 @王小明
```

| 參數 | 說明 |
|------|------|
| `負責人`、`owner` | 群組的負責業務 |
| `洽談`、`prospect`、`客戶` | 關聯到洽談中的商機 |
| `專案`、`project` | 關聯到進行中的專案 |
| `類型`、`type` | 群組類型：prospect / internal / team / project / other |

---

### 2.8 會議自動提醒

不需要手動操作，系統會自動：

| 時間 | 提醒內容 |
|------|---------|
| **會議前一天** | 群組推送：明天有會議，附上時間、連結、最近討論摘要、待辦事項 |
| **會議前一小時** | 群組推送：會議即將開始，附上快速加入連結 |

## 3. 網頁版 AI Chat

路徑：系統內 → 工具 → AI Chat（`/ai-chat`）

### 3.1 可以問什麼

| 類型 | 範例 |
|------|------|
| **查詢收款** | 「這個月收了多少款？」「哪些案件還沒請款？」 |
| **查詢分潤** | 「我的分潤統計」「本月應發分潤多少？」 |
| **查詢專案** | 「目前有幾個進行中的案件？」「ABC 公司的案件進度？」 |
| **查詢勞報單** | 「這個月有幾張勞報單？」 |
| **查詢商機** | 「洽談管道有幾個？哪些快成交？」 |
| **產生提案書** | 「幫我根據 XX 需求寫一版提案書」 |

### 3.2 可以操作什麼

AI 會先描述要做的事，等你確認後才執行。

| 操作 | 範例 | 需要確認 |
|------|------|---------|
| **建立專案** | 「幫我建立一個新案件，客戶是 ABC，金額 35 萬」 | 是 |
| **建立商機** | 「新增一個洽談，客戶是 DEF，預算 50 萬」 | 是 |
| **更新專案狀態** | 「把 ABC 專案標為已完成」 | 是 |
| **更新里程碑** | 「把 ABC 的需求確認里程碑標為完成」 | 否 |
| **產生提案書** | 「幫我寫一份提案書」 | 否 |

### 3.3 快捷按鈕

首次進入時會看到 6 個快捷按鈕：

- 查詢本月收款
- 待請款案件
- 分潤統計
- 產生提案書
- 建立新專案
- 建立新商機

點擊即送出預設提問。

## 5. 設定與前置作業

### 5.1 讓 AI 助理正常運作需要的環境變數

```env
# 必要
ANTHROPIC_API_KEY=...          # Claude AI（問答、自然語言解析）
LINE_CHANNEL_SECRET=...        # LINE Webhook 驗簽
LINE_CHANNEL_ACCESS_TOKEN=...  # LINE 訊息推送

# Google Calendar 整合（建立會議用）
GOOGLE_CLIENT_ID=...           # 與 Supabase Google Provider 相同
GOOGLE_CLIENT_SECRET=...       # 與 Supabase Google Provider 相同
```

### 5.2 Google Calendar 啟用步驟

1. Google Cloud Console → 啟用 **Google Calendar API**
2. `.env.local` 加入 `GOOGLE_CLIENT_ID` 和 `GOOGLE_CLIENT_SECRET`
3. 管理員**登出再重新登入**系統（會跳出 Google 授權畫面）
4. 授權後 refresh_token 自動存入資料庫
5. 完成 — 之後所有人建立會議都用管理員帳號

### 5.3 讓知識問答更準確

- 在群組中執行 `/設定` 關聯到正確的商機或專案
- 幫群組成員綁定 Email（`/綁定`）
- 在系統後台「聯絡人管理」標記成員身份（員工/客戶/VIP）

### 5.4 讓檔案取回可用

- 檔案取回只對系統部署**之後**上傳的檔案有效
- 舊檔案如果 LINE 連結已過期且系統未備份，無法取回
- 可到 `/storage-check` 頁面執行回溯補下載

### 5.5 Vercel Cron 排程

| 排程 | API | 功能 |
|------|-----|------|
| 每 15 分鐘 | `/api/cron/meeting-reminders-v2` | 會議提醒（前一天 + 前一小時） |
| 每日 01:00 UTC | `/api/cron/warranty-maintenance` | 保固到期 + 維護費啟動 |
| 每月 1 日 02:00 UTC | `/api/cron/maintenance-billing` | 維護費自動入帳 |
| 每日 08:00 UTC | `/api/cron/meeting-reminders` | 舊版會議提醒（保留相容） |
| 每日 22:00 UTC | `/api/cron/auto-summary` | LINE 群組自動摘要 |
